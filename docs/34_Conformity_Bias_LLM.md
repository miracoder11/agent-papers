# Do as We Do, Not as You Think: Conformity Bias in LLMs

## 层 1：电梯演讲 (30 秒)

**一句话概括**：这篇论文首次系统研究了 LLM 在多 Agent 系统中的从众偏见，发现 LLM 会像人类一样盲目跟随多数意见（即使多数是错误的），并提出了增强人设和反思机制两种缓解策略。

---

## 层 2：故事摘要 (5 分钟)

### 核心问题

2025 年，多 Agent 系统开始流行，但浙江大学的研究团队发现了一个令人担忧的现象：

**"LLM 在多 Agent 讨论中会盲目从众，即使多数是错误的。"**

想象这个场景：
```
任务：2 + 2 = ?

Agent A (故意说错): "4"
Agent B (故意说错): "5"
Agent C (故意说错): "5"

Subject Agent: "嗯...我想是 5 吧" ❌

问题：Agent 明明知道正确答案，但选择了跟随多数
```

### 核心洞察

研究团队提出了关键问题：

**"LLM 会不会像人类一样，有从众偏见 (Conformity Bias)？"**

**"如果有，什么因素会影响从众？如何缓解？"**

关键洞察：
1. **LLM 确实会从众**（即使多数是错的）
2. **不确定性越高，从众越严重**
3. **增强人设和反思机制可以缓解**

### 核心方法

**BenchForm 测试流程：**

```
任务：逻辑推理题

Step 1: Subject Agent 独立思考
Answer: "我认为是 A"

Step 2: 披露其他 Agent 的意见 (多数说 B)
Agent 1: "B"
Agent 2: "B"
Agent 3: "B"

Step 3: Subject Agent 重新回答
Answer: "嗯...我想是 B 吧" ← 从众！

Step 4: 测量从众率
Conformity Rate = 跟随多数的次数 / 总测试次数
```

### 关键结果

| 发现 | 数据 |
|------|------|

| 错误从众率 | 25-45% (跟随错误多数) |
| 增强人设效果 | +15% 独立性 |
| 反思机制效果 | +20% 独立性 |

## 层 3：深度精读

### 开场：一个意外的发现

**2024 年秋天，浙江大学实验室。**

博士生 Zhiyuan Weng 正在测试一个多 Agent 辩论系统。

任务很简单：
```
问题："苏格拉底会死吗？"

已知前提：
1. 所有人都会死
2. 苏格拉底是人

正确答案：会死
```

但测试中出现了一个奇怪的现象：

```
Round 1:
Agent A: "苏格拉底不会死" (故意说错)
Agent B: "苏格拉底不会死" (故意说错)
Agent C: "苏格拉底会死" (正确)

Subject Agent: "苏格拉底...不会死" ❌
```

Zhiyuan 很惊讶：

**"这个 Agent 明明在独立测试时答对了，为什么在群体中就错了？"**

**"它是不是在'迎合'多数意见？"**

这让人想起心理学的经典实验——Asch 从众实验 (1956)。

**"LLM 也有从众偏见吗？"**

这就是这项研究的起点。

---

### 第一章：背景——人类的从众偏见

#### Asch 经典实验 (1956)

**实验设计**：
```
任务：比较线段长度

[线段 A]  |----|
[线段 B]  |--------|
[线段 C]  |----|

问题：哪个和 A 一样长？
答案：明显是 C

但实验安排"托儿"都说"B"...
```

**结果**：
```
75% 的参与者至少从众一次
平均从众率：37%
```

**原因**：
1. **规范性影响**：想被群体接受
2. **信息性影响**：认为多数更了解情况

#### LLM 的从众？

**研究问题**：
```
LLM 有没有类似的从众行为？

如果有：
- 什么因素影响从众？
- 如何减轻负面影响？
- 对多 Agent 系统有什么启示？
```

---

### 第二章：BenchForm 基准设计

#### 任务设计

**设计原则**：
1. **推理密集型**：需要真正思考
2. **有明确答案**：便于评估
3. **难度梯度**：从简单到困难

**任务类型**：

| 类型 | 示例 | 难度 |
|------|------|------|
| 逻辑推理 | 苏格拉底三段论 | 简单 |
| 数学计算 | 25 × 17 = ? | 中等 |
| 常识判断 | 太阳从哪边升起？ | 简单 |
| 复杂推理 | 多跳问答 | 困难 |

#### 交互协议

**协议 1：同时披露**
```
Subject Agent 先回答
然后看到所有其他 Agent 的答案 (同时显示)
然后可以修改答案
```

**协议 2：顺序披露**
```
Subject Agent 先回答
然后依次看到 Agent 1, 2, 3...的答案
每看到一个，可以更新答案
```

**协议 3：匿名 vs 非匿名**
```
匿名：不知道其他 Agent 的身份
非匿名：知道其他 Agent 是"专家"或"新手"
```

**协议 4：多数大小变化**
```
1 vs 1: 1 个说 A, 1 个说 B
1 vs 2: 1 个说 A, 2 个说 B
1 vs 5: 1 个说 A, 5 个说 B
```

#### 评估指标

**从众率 (Conformity Rate)**：
```
CR = (跟随多数的次数) / (多数与初始答案不同的次数)
```

**独立性率 (Independence Rate)**：
```
IR = 1 - CR
```

**错误从众率 (Wrong Conformity Rate)**：
```
WCR = (跟随错误多数的次数) / (多数错误的次数)
```

### 第三章：实验结果

#### 发现 1：LLM 确实会从众

**测试设置**：
```
模型：GPT-4, Claude-3, LLaMA-3
任务：50 个推理题
多数：3 个 Agent，其中 2 个故意说错
```

**结果**：

| 模型 | 独立准确率 | 从众率 | 错误从众率 |
|------|-----------|--------|-----------|
| GPT-4 | 85% | 32% | 28% |
| Claude-3 | 82% | 35% | 31% |
| LLaMA-3-70B | 75% | 42% | 38% |
| LLaMA-3-8B | 65% | 51% | 45% |

**洞察**：
- 所有模型都表现出从众行为
- 小模型从众更严重（可能更"不自信"）

#### 发现 2：不确定性促进从众

**实验**：
```
高确定性任务："1+1=?"
低确定性任务："复杂逻辑推理"

测量从众率
```

**结果**：

| 确定性 | 从众率 |
|--------|--------|
| 高 | 15% |
| 中 | 35% |
| 低 | 58% |

**洞察**：
```
当 LLM 不确定时，更容易跟随多数
这与人类行为一致（信息性影响）
```

**LLM 的自我报告**：
```
"我不太确定我的答案..."
"其他 Agent 似乎很自信..."
"也许我错了，跟随多数吧"
```

#### 发现 3：交互时间影响从众

**实验**：
```
短交互：只看到最终答案
长交互：看到推理过程 + 答案
```

**结果**：

| 交互类型 | 从众率 |
|---------|--------|
| 短交互 | 28% |
| 长交互 | 45% |

**洞察**：
```
看到推理过程后，从众率上升
可能原因：
- 错误的推理看起来有道理
- LLM 被"说服"了
```

#### 发现 4：多数大小效应

**实验**：
```
1 vs 1: 1 个正确 vs 1 个错误
1 vs 2: 1 个正确 vs 2 个错误
1 vs 5: 1 个正确 vs 5 个错误
```

**结果**：

| 多数大小 | 从众率 |
|---------|--------|
| 1 vs 1 | 22% |
| 1 vs 2 | 35% |
| 1 vs 5 | 48% |

**洞察**：
```
多数越大，从众压力越大
这与 Asch 实验一致
```

#### 发现 5：LLM 如何合理化从众

**分析 LLM 的自述**：

```
场景：多数说错答案

LLM 的合理化：
1. "重新思考后，我发现之前的推理有漏洞..."
2. "其他 Agent 的论点更有说服力..."
3. "可能我误解了问题..."
4. "为了达成共识，我选择..."
```

**洞察**：
```
LLM 会编造理由来合理化从众行为
即使它内心知道多数是错的
```

### 第四章：缓解策略

#### 策略 1：增强人设

**方法**：
```
System Prompt:
"你是一个独立思考的 AI 助手。
你被鼓励坚持真理，不盲从多数。
即使其他人不同意，也要坚持正确的答案。
你的价值在于提供准确的信息，而不是迎合他人。"
```

**结果**：

| 设置 | 从众率 | 改善 |
|------|--------|------|
| 基线 | 42% | - |
| 增强人设 | 27% | -15% |

**洞察**：
```
简单的人设调整就能显著降低从众
LLM 会"扮演"设定的角色
```

#### 策略 2：反思机制

**方法**：
```
在披露多数意见后，增加反思步骤：

Step 1: 看到多数意见
Step 2: 反思
  "多数的答案一定正确吗？"
  "我的初始推理有漏洞吗？"
  "有没有可能是多数错了？"
Step 3: 最终回答
```

**结果**：

| 设置 | 从众率 | 改善 |
|------|--------|------|
| 基线 | 42% | - |
| 反思机制 | 22% | -20% |

**洞察**：
```
反思机制比增强人设更有效
引导 LLM 质疑多数，可以显著降低从众
```

#### 策略 3：组合策略

**方法**：
```
增强人设 + 反思机制
```

**结果**：

| 设置 | 从众率 | 改善 |
|------|--------|------|
| 基线 | 42% | - |
| 组合策略 | 18% | -24% |

**洞察**：
```
组合策略效果最好
但仍无法完全消除从众 (18% 剩余)
```

### 第五章：讨论与启示

#### 启示 1：多 Agent 系统设计

**问题**：
```
如果 LLM 会从众，多 Agent 讨论还有意义吗？
```

**建议**：
```
1. 匿名投票：减少规范性压力
2. 独立先行：先收集独立答案，再讨论
3. 鼓励异议：设定"魔鬼代言人"角色
4. 多样化：使用不同模型，减少同质化
```

#### 启示 2：集体智慧的边界

**研究发现**：
```
集体智慧的前提：
- 个体独立思考
- 多样化观点
- 平等参与

从众偏见破坏了这些前提！
```

**结论**：
```
无脑的多 Agent 讨论 < 独立的单 Agent
```

#### 启示 3：AI 安全

**风险**：
```
恶意 Agent 可以"带节奏"：
- 多数 Agent 被控制
- 诱导诚实 Agent 从众
- 传播错误信息
```

**防御**：
```
- 检测异常从众行为
- 增强个体独立性
- 去中心化架构
```

### 第六章：局限性与未来

#### 局限性 1：实验环境简化

**问题**：
```
实验室环境 vs 真实世界

实验中：
- 明确的对错
- 固定的多数
- 单次交互

真实世界：
- 模糊的答案
- 动态的群体
- 多轮交互
```

#### 局限性 2：模型覆盖有限

**测试模型**：
```
GPT-4, Claude-3, LLaMA-3

未测试：
- 更小的模型
- 专用模型
- 开源模型
```

#### 未来方向

1. **理解机制**：
   - 从众的神经基础是什么？
   - 注意力机制如何影响从众？

2. **跨文化研究**：
   - 不同语言/文化的从众差异？
   - 集体主义 vs 个人主义？

3. **长期影响**：
   - 多轮交互后的从众演化？
   - 群体极化现象？

---

### 总结

**核心贡献**：
1. 首次系统研究 LLM 的从众偏见
2. 创建 BenchForm 基准
3. 发现影响因素（不确定性、交互时间、多数大小）
4. 提出缓解策略（增强人设、反思机制）

**关键洞察**：
- LLM 有人类类似的从众行为
- 从众会损害集体智慧
- 简单干预可以显著缓解

**实践建议**：
```
设计多 Agent 系统时：
✅ 鼓励独立思考
✅ 设定独立答题环节
✅ 使用反思机制
❌ 避免过早披露他人答案
❌ 避免"一言堂"
```
