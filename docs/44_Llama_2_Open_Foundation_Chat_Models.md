### 第三章：核心概念 - 大量实例

#### 3.1 分组查询注意力 (GQA)

**生活类比 1：图书管理员**

想象一个大型图书馆：

```
传统 MHA (Multi-Head Attention)：
- 每个读者（query）配一个专属管理员（key/value）
- 100 个读者 = 100 个管理员
- 效率低，资源浪费

GQA (Grouped-Query Attention)：
- 多个读者共享一个管理员
- 100 个读者 = 8 个管理员
- 效率大幅提升，效果几乎不变
```

**代码示例**

```python
# MHA: 每个 head 有自己的 Q, K, V
class MHA(nn.Module):
    def __init__(self, dim, n_heads):
        self.wq = nn.Linear(dim, dim)
        self.wk = nn.Linear(dim, dim)
        self.wv = nn.Linear(dim, dim)
        # n_heads 个独立的注意力头

# GQA: query 分组，每组共享 K, V
class GQA(nn.Module):
    def __init__(self, dim, n_heads, n_kv_heads):
        self.wq = nn.Linear(dim, n_heads * head_dim)
        self.wk = nn.Linear(dim, n_kv_heads * head_dim)  # 更少的 KV
        self.wv = nn.Linear(dim, n_kv_heads * head_dim)
        # n_heads > n_kv_heads，query 分组共享 KV
```

**性能对比**

```
70B 模型推理性能（A100 GPU）：
┌─────────────────────────────────────────┐
│ MHA:   15 tokens/sec, 48GB KV cache     │
│ GQA:   28 tokens/sec, 12GB KV cache     │
│                                         │
│ 提升：1.87 倍速度，75% 内存节省           │
└─────────────────────────────────────────┘
```

#### 3.2 RLHF 奖励建模

**生活类比 2：美食评委**

```
想象训练一个美食评委：

阶段 1（数据收集）：
- 给评委吃很多菜品对
- 问："这道好吃还是那道好吃？"
- 记录评委的选择

阶段 2（学习标准）：
- 评委学习什么是"好吃"
- 咸淡、口感、香气...
- 形成内在评价标准

阶段 3（指导厨师）：
- 厨师做菜，评委打分
- 厨师根据分数调整
- 反复迭代，菜越来越好

RLHF 完全相同：
- 奖励模型 = 美食评委
- 语言模型 = 厨师
- 人类偏好标注 = 评委试吃
```

**奖励模型训练示例**

```python
# 奖励模型输入输出
输入：{
    "prompt": "如何学习机器学习？",
    "response": "学习机器学习建议..."
}
输出：{
    "reward_score": 0.87  # 越高越好
}

# 训练数据示例
训练样本 = [
    {
        "prompt": "...",
        "chosen": "人类选择的回答",  # 奖励应该高
        "rejected": "人类拒绝的回答"  # 奖励应该低
    },
    ...
]

# 损失函数
loss = -log(sigmoid(reward_chosen - reward_rejected))
# 目标：chosen 的奖励比 rejected 高
```

#### 3.3 多轮对话一致性

**问题场景**

```
对话开始：
用户：我叫小明，今年 25 岁，在北京工作
助手：你好小明！很高兴认识你...

第 5 轮：
用户：你还记得我叫什么吗？
助手：抱歉，我不记得了...（糟糕！）

第 10 轮：
用户：我之前说我在哪工作？
助手：你没有告诉过我...（更糟糕！）
```

**Llama 2 的解决方案：System Message**

```python
# 对话格式
conversation = [
    {"role": "system", "content": "你是 Llama 2，一个友好的 AI 助手。"},
```

### 第四章：预期 vs 实际

#### 4.1 直觉预测 vs 实际实现

**预期 1：更大的模型 = 更好的效果**

| 你的直觉 | 实际 | 为什么 |
|---------|------|--------|
| 70B 应该远超 13B | 在某些任务上差距很小 | 数据质量和训练方式更重要 |
| 参数越多越好 | 34B 效果接近 70B | 存在规模收益递减 |
| 大模型全能 | 小模型在特定任务可能更好 | 需要针对任务选择 |

**预期 2：微调会损害通用能力**

| 你的直觉 | 实际 | 为什么 |
|---------|------|--------|
| 对话微调后不会写代码了 | 通用能力保持良好 | 多样化微调数据 |
| 学对话忘知识 | 知识保留 + 对话能力 | 微调数据量相对小 |
| 需要多个版本 | 一个模型多用途 | 微调不破坏基础能力 |

**预期 3：安全对齐会降低有用性**

| 你的直觉 | 实际 | 为什么 |
|---------|------|--------|
| 安全=什么都不回答 | 只在真正有害时拒绝 | 精细的安全边界 |
| 安全模型很无趣 | 依然可以有趣有用 | 安全≠无聊 |
| 越狱很容易 | 需要复杂攻击 | 多层次防护 |

#### 4.2 反直觉挑战

**问题：如果在训练数据中加入更多安全内容，模型会更安全吗？**

[先想 1 分钟...]

**直觉**：当然会！学得越多越好。

**实际**：不一定！关键在于如何教，而不是教多少。

```
错误方式：
- 大量有毒数据 + 少量安全标注 = 模型还是有风险
- 因为模型先学习了有毒模式

正确方式：
- 过滤有毒训练数据
- 高质量安全微调数据
- 清晰的安全边界教学
```

**Llama 2 的做法**：
- 预训练阶段就过滤有害数据
- 安全微调用精心设计的对话
- 红队测试发现边界情况

### 第五章：关键实验的细节

#### 5.1 人类评估实验设计

**评估指标**

```
1. 有用性 (Helpfulness)

   - 是否满足用户需求

   - 是否输出有害内容
   - 是否拒绝不当请求
   - 是否符合伦理标准

3. 偏好 (Preference)
   - A/B 测试胜率
   - 用户更喜欢哪个回答
```

**评估流程**

```
步骤 1：收集测试 prompt
- 4000+ 个提示词

- 包含单轮和多轮

- 所有候选模型生成
- 匿名化处理
- 随机排序

步骤 3：人类标注
- 专业标注员评估
- 盲测（不知道是哪个模型）
- 多维度打分

步骤 4：统计分析
- 计算胜率
- 置信区间
- 显著性检验
```

#### 6.2 局限性分析

**Llama 2 的局限**

```
1. 英语为主
   - 其他语言能力较弱
   - 训练数据主要是英文

2. 知识截止
   - 训练数据截止到 2023 年 7 月
   - 不知道之后发生的事

3. 推理能力有限
   - 复杂数学和逻辑推理不如专用模型
   - 需要额外训练

4. 长上下文处理
   - 4K 上下文比 Claude 的 100K 短很多
   - 处理长文档有困难

5. 需要申请
   - 不是完全开源
   - 商用需要审批
```

#### 6.3 改进方向

```
1. 多语言支持
   - 增加非英语训练数据
   - 针对特定语言微调

2. 更长上下文
   - 扩展到 32K+
   - 需要更高效的注意力机制

3. 推理能力
   - 数学和代码专项训练
   - 结合符号推理

4. 多模态
   - 图像理解
   - 音频处理

5. 持续学习
   - 知识更新机制
   - 避免灾难性遗忘
```

```python
# 使用 HuggingFace Transformers
from transformers import AutoTokenizer, AutoModelForCausalLM

# 加载模型
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-chat-hf")
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-chat-hf")

# 对话
prompt = "你好，请介绍一下你自己"
inputs = tokenizer(prompt, return_tensors="pt")
outputs = model.generate(**inputs, max_new_tokens=100)
print(tokenizer.decode(outputs[0]))
```
