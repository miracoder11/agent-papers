# WebGPT: Browser-based Question-Answering with Human-Feedback

## 层 1：电梯演讲

**一句话概括**：OpenAI 团队在 2021 年发现 LLM 在长形式问答任务上容易产生幻觉，通过构建文本浏览器环境和人类反馈优化流程，训练出的 WebGPT 模型能够自主浏览网页搜集信息，在答案质量上超越人类演示者（56% 偏好率）和 Reddit 高赞回答（69% 偏好率），大幅减少幻觉。

## 层 2：故事摘要

### 核心问题

2021 年的 LLM 研究面临**根本性挑战**：

**LLM 的幻觉困境**：
```
用户：\"新冠疫苗的长期副作用有哪些？\"

标准 GPT-3 回答：
\"根据研究，新冠疫苗可能导致...
[开始编造数据和研究]...\"

问题：
- 看起来很有说服力
- 但实际上是编造的
- 用户无法验证信息来源
```

**人类的优势**：
```
人类专家回答同样的问题：
1. 搜索\"新冠疫苗 长期副作用 site:cdc.gov\"
2. 打开 CDC 官方页面
3. 滚动查找相关信息
4. 引用具体数据
5. 标注信息来源

结果：可验证、可信
```

### 核心洞察

OpenAI 团队问了一个关键问题：

> \"如果让 LLM 学会像人类一样上网搜索，会怎样？\"

**关键观察**：
1. 人类遇到不知道的问题 → 上网搜索 → 整理信息 → 回答
2. GPT-3 遇到不知道的问题 → 编造答案 → 看起来合理
3. 差距不在语言能力，而在**信息搜集能力**

**答案是**：WebGPT 诞生了，LLM 学会使用浏览器作为工具。

## 层 3：深度精读

### 开场：一个\"简单\"的问题

**时间**：2021 年
**地点**：OpenAI 办公室
**人物**：Reiichiro Nakano, Jacob Hilton 团队

**场景**：
```
研究员问 GPT-3 一个问题：

问题：\"2020 年美国大选各州的投票率是多少？\"

GPT-3 回答：
\"2020 年大选中，加州的投票率约为 80%，
德州约为 65%，佛罗里达约为 75%...\"

看起来很有说服力，对吧？

但问题是：这些数据是编造的。

实际数据需要查询官方来源，但 GPT-3 做不到。
它只能从训练数据中\"回忆\"，无法\"查找\"真实信息。

研究员的困惑：
\"人类遇到这个问题会怎么做？\"

答案：打开浏览器，搜索，查官方数据。

\"那为什么不让模型也学会上网呢？\"

这就是 WebGPT 的起点。
```

### 第一章：2021 年 LLM 的三大困境

#### 困境 1：无法获取最新信息

**训练数据的截止时间**：
```
GPT-3 训练数据截止：2021 年之前

用户 2022 年问：\"最新发布的 GPT-4 有什么特点？\"

GPT-3: \"我没有关于 GPT-4 的信息...\"

问题：世界在变，模型的知识是静态的
```

**人类的解决方案**：
```
人类：打开 Google → 搜索\"GPT-4 特点\" → 找到官方博客 → 阅读

结果：获取最新信息
```

#### 困境 2：幻觉问题

**具体案例**：
```
问题：\"引用一篇关于气候变化对北极熊影响的论文\"

标准 GPT-3:
\"根据 Smith et al. (2019) 在 Nature 发表的
'Climate Impact on Polar Bears'...
[引用格式完美，但这篇论文不存在]\"

问题：模型编造引用，用户无法验证
```

#### 困境 3：长形式问答的挑战

**ELI5 数据集的挑战**：
```
问题示例（来自 r/ExplainLikeImFive）：

\"既然黑洞连光都吸，为什么宇宙还在膨胀？\"

这需要：
1. 理解问题（黑洞 vs 宇宙膨胀）
2. 搜集信息（查物理知识）
3. 组织成长篇解释（500+ 词）
4. 标注来源（让读者验证）

标准 GPT-3 的问题：
- 不查资料，靠训练数据硬编
- 看起来合理，但可能错误
- 没有来源，无法验证
```

### 第二章：核心洞察 - 浏览器作为工具

#### 关键观察

**观察 1：人类如何回答问题**：
```
实验：让研究员回答\"新冠疫苗的长期副作用\"

行为记录：
1. 打开 Google (5 秒)
2. 搜索\"新冠疫苗 长期副作用 site:cdc.gov\" (10 秒)
3. 点击 CDC 官方链接 (3 秒)
4. 滚动页面查找相关信息 (15 秒)
5. 复制关键数据 (5 秒)
6. 再搜索一篇研究论文 (30 秒)
7. 整合信息，写答案 (120 秒)

总时间：~3 分钟
关键：每一步都在搜集和验证信息
```

**观察 2：LLM 能学会这些动作吗？**
```
如果把浏览器的交互简化为文本：

网页 = 文本（标题 + 链接 + 内容）
动作 = 文本命令（搜索、点击、滚动）

LLM 擅长处理文本！

所以...LLM 应该能学会浏览网页？
```

#### 设计原则

**原则 1：文本化**
```
网页有图片、样式、交互...太复杂了！

简化方案：
- 只保留文本
- 链接用 [id] 标注
- 层级用缩进表示

示例：
┌─ 页面标题
│  页面内容...
├─ [1] 链接 1
├─ [2] 链接 2
└─ [3] 链接 3

LLM：\"我看懂了，这是文本！\"
```

**原则 2：原子化动作**
```
人类浏览网页的动作看似复杂，但可以分解：

搜索 → search[keywords]
点击链接 → click[id]
页内查找 → find_in_page[text]
引用内容 → quote[...]
滚动页面 → scroll_up/down
回顶部 → top
返回 → back
结束 → end

8 个原子动作，覆盖所有浏览行为！
```

**原则 3：人类示范**
```
与其让模型从零摸索，不如直接教它：

1. 请人类演示如何浏览网页回答问题
2. 记录每一步动作和最终答案
3. 用这些数据训练模型

\"看，人类是这样做的！\"
```

### 第三章：文本浏览器环境

#### 环境设计

**网页的文本表示**：
```python
# 原始 HTML
<html>
  <h1>新冠疫苗副作用</h1>
  <p>根据 CDC 数据，常见副作用包括...</p>
  <a href="...">了解更多</a>
</html>

# WebGPT 的文本表示
= 新冠疫苗副作用

根据 CDC 数据，常见副作用包括：
- 注射部位疼痛
- 疲劳
- 头痛
- 肌肉疼痛

[1] 了解更多：https://cdc.gov/vaccine-side-effects
[2] 相关研究：https://nih.gov/covid-studies
```

**8 种动作详解**：

| 动作 | 功能 | 示例 |
|------|------|------|
| `search[...]` | 用搜索引擎搜索 | `search[新冠疫苗 副作用]` |
| `click[...]` | 点击链接 | `click[5]` |
| `find_in_page[...]` | 页内查找 | `find_in_page[长期]` |
| `quote[...]` | 引用内容 | `quote[常见副作用包括...]` |
| `scroll_up` | 向上滚动 | `scroll_up` |
| `scroll_down` | 向下滚动 | `scroll_down` |
| `top` | 回到顶部 | `top` |
| `back` | 返回上一页 | `back` |

### 第四章：数据收集与训练

#### 人类演示数据

**数据规模**：
```
- 演示轨迹：~6,000 条
- 每条评价：包含问题、浏览历史、答案、引用
- 问题来源：ELI5 (Explain Like I'm 5)
- 演示者：经过培训的标注人员

时间成本：
- 每条演示：5-10 分钟
- 总时间：~500 小时
- 成本：约$30,000
```

**数据质量要求**：
```
好的演示标准：
1. 搜索关键词准确
2. 选择的来源可靠（.gov, .edu, 知名媒体）
3. 引用内容完整
4. 答案组织清晰
5. 标注所有信息来源

坏的演示（需要返工）：
- 随便点击，没有目的
- 引用不可靠来源
- 答案没有引用
- 复制粘贴而非整合
```

#### 人类偏好数据

**偏好对比收集**：
```
方法：
1. 对同一个问题，生成多个答案
2. 请人类评分哪个答案更好
3. 收集约 21,500 对偏好

评分标准：
- 答案是否准确
- 引用是否可靠
- 组织是否清晰
- 是否完整回答问题
```

**示例对比**：
```
问题：\"为什么天空是蓝色的？\"

答案 A:
\"因为瑞利散射。阳光中的蓝光波长较短，
更容易被大气分子散射。[引用 Physics Today]\"
评分：★★★★★

答案 B:
\"天空是蓝色的因为海洋反射...\"
评分：★★☆☆☆

人类偏好：A >> B
```

---

#### 两阶段训练

**阶段 1：行为克隆（Imitation Learning）**
```
输入：问题 + 初始浏览器状态
输出：动作序列 + 最终答案

训练目标：
- 预测人类的动作（search, click, quote...）
- 预测人类的答案

损失函数：
L = L_actions + L_answer

效果：
- 模型学会基本的浏览动作
- 知道什么时候搜索、点击、引用
- 但质量还不够高
```

**阶段 2：强化学习（RLHF）**
```
Step 1: 训练奖励模型
- 输入：答案 + 引用
- 输出：人类偏好分数
- 训练数据：21,500 对偏好

Step 2: PPO 优化策略
- 用奖励模型给答案评分
- 最大化期望奖励
- 同时保持语言流畅

效果：
- 答案质量大幅提升
- 更可靠的引用
- 更好的组织结构
```

### 反直觉挑战

**问题：如果去掉 RLHF，只用行为克隆，会怎样？**

[先想 1 分钟...]

**直觉**：\"应该差不多吧？毕竟学会了人类的操作\"

**实际**：
```
行为克隆 alone:
- 答案质量：明显较差
- 引用可靠性：低
- 人类偏好率：~35%

行为克隆 + RLHF:
- 答案质量：高
- 引用可靠性：高

差距 21%！
```

**为什么**：
- 行为克隆只学会\"模仿动作\"
- RLHF 学会\"什么是好的答案\"
- 模仿 ≠ 优秀
- 需要人类反馈告诉模型什么是高质量

### 第八章：局限性与改进

#### 局限性

**1. 只能处理文本**
```
问题：
- 无法处理图片、视频
- 如果答案在图表里，会错过
- 网页布局复杂时可能解析错误

影响:
- 某些任务无法完成（如\"分析这张图表\"）
- 信息获取受限
```

**2. 浏览深度有限**
```
观察:
- WebGPT 通常浏览 3-5 个页面
- 很少超过 10 步

问题:
- 复杂问题需要深入研究
- 可能被浅层信息误导
- 不会\"深挖\"关键来源
```

**3. 无法跨会话学习**
```
问题:
- 每次浏览都是独立的
- 不会记住之前的成功经验
- 第二次回答同样问题不会比第一次好

对比人类:
- 人类会记住\"这个来源可靠\"
- 会积累经验
```

**4. 依赖人类反馈质量**
```
问题:
- 如果人类评分者判断错误，模型会学到错误
- 人类评分者可能有偏见
- 某些领域（如专业科学）评分者不够专业
```

#### 改进方向

**1. 多模态扩展**
```
未来方向:
- 添加图片理解能力
- 处理图表、数据可视化
- 甚至处理视频内容

技术路径:
- 集成视觉模型（如 CLIP）
- 多模态 transformer
```

**2. 长期记忆**
```
改进方案:
- 添加记忆模块
- 记住可靠来源
- 记住成功的浏览策略

效果:
- 越用越聪明
- 积累领域知识
```

**3. 自我反思**
```
集成 Reflexion:
- 浏览后自我评估：\"我找到足够的信息了吗？\"
- 如果不够，继续搜索
- 从失败的浏览中学习
```

**4. 多 Agent 协作**
```
想法:
- Agent 1: 负责搜索
- Agent 2: 负责验证来源
- Agent 3: 负责整合答案

好处:
- 分工合作，质量更高
- 互相校验，减少错误
```

**步骤 1：构建文本浏览器**
```python
import requests
from bs4 import BeautifulSoup

def webpage_to_text(url):
    \"\"\"将网页转换为文本\"\"\"
    html = requests.get(url).text
    soup = BeautifulSoup(html, 'html.parser')

    # 提取标题
    title = soup.find('title').text if soup.find('title') else 'Unknown'

    # 提取链接
    links = []
    for i, a in enumerate(soup.find_all('a', href=True)):
        links.append(f"[{i+1}] {a.text}: {a['href']}")

    # 提取正文
    text = soup.get_text(separator='\\n', strip=True)

    return f\"= {title}\\n\\n{text}\\n\\nLinks:\\n\" + '\\n'.join(links)
```
