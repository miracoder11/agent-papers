# Attention Is All You Need (Transformer)

## 层 1：电梯演讲

**一句话概括**：Google 团队在 2017 年提出完全基于注意力机制的 Transformer 架构，摒弃 RNN 和 CNN，在机器翻译任务上超越所有现有模型，训练时间从几天缩短到几小时，开启了大语言模型时代。

---

## 层 2：故事摘要

### 核心问题
2017 年前的序列建模被 RNN/LSTM 主导，但存在**根本性局限**：
- **序列计算**：必须按顺序处理，无法并行
- **长程依赖**：距离远的信息难以传递
- **训练缓慢**：无法充分利用 GPU

### 核心洞察
Google Brain 团队问了一个简单但深刻的问题：
> "如果只用注意力机制，不要 RNN 和 CNN，会怎样？"

**答案是**：Transformer 诞生了，效果空前的好。

### 第一章：RNN 的局限

#### RNN 的工作原理

**RNN 计算**：
```
h₁ = RNN(x₁, h₀)
h₂ = RNN(x₁, h₁)
h₃ = RNN(x₃, h₂)
...

必须按顺序计算！
h₂ 依赖 h₁，h₃ 依赖 h₂...
```

**问题 1：无法并行**
```
GPU 喜欢并行计算，但 RNN 必须串行：

时间步 1: [GPU 工作]
时间步 2: [GPU 工作]
时间步 3: [GPU 工作]
...

GPU 利用率低，训练慢。
```

**问题 2：长程依赖**
```
句子："我出生在法国...（100 个词后）...所以我说法语"

RNN 需要记住"法国"来选择"法语"。
但 100 步后，信息已经模糊了。

LSTM 缓解了，但没根本解决。
```

#### 注意力机制的兴起

**注意力的核心思想**：
```
传统 RNN：
- 信息必须通过隐藏状态传递
- 距离越远，信息越少

注意力：
- 直接"看"任何位置
- 距离不影响信息获取

例：
"我住在法国...（很多词）...我喜欢在巴黎生活"

注意力让"巴黎"直接关注"法国"，不管距离。
```

**但 2017 年前的注意力**：
```
注意力只是 RNN 的"配件"：
Encoder (RNN) → 表示
Decoder (RNN) + Attention → 输出

没人想过：只用注意力，不要 RNN？
```

---

### 第二章：Transformer 的诞生

#### 关键洞察

**2017 年，Google Brain 团队问**：
> "如果完全不用 RNN，只用注意力，会怎样？"

**挑战**：
1. 如何捕捉序列顺序？（没有 RNN 的时序）
2. 如何并行计算？
3. 效果能比 RNN 好吗？

#### Self-Attention 机制

**核心公式**：
```
Attention(Q, K, V) = softmax(QK^T / √d_k) V

Q: Query（查询）
K: Key（键）
V: Value（值）
```

**直观理解**：
```
想象一个图书馆：

Q: 你想找的书（查询）
K: 每本书的标签（键）
V: 书的内容（值）

计算 Q 和每个 K 的相似度 → 权重
用权重加权 V → 最终结果

例：
句子："The animal didn't cross the street because it was too tired"

"it" 的 Q 会关注：
- "animal"（高权重）
- "street"（低权重）

这样就知道"it"指代"animal"。
```

#### Scaled Dot-Product Attention

**为什么需要 Scaling（除以 √d_k）？**

**问题**：
```
当 d_k 很大时，点积结果也很大。

例：
q · k = Σ q_i * k_i

如果 d_k=512，点积可能很大（如 1000）。
softmax(1000) → 梯度几乎为 0。
```

**解决**：
```
除以 √d_k：
(q · k) / √d_k

这样梯度不会消失。
```

#### Multi-Head Attention

**核心思想**：
```
与其做一个注意力，不如做多个并行的注意力头。

例：
句子："The animal didn't cross the street because it was too tired"

Head 1: "it" → "animal"（语法关系）
Head 2: "it" → "tired"（语义关系）
Head 3: "it" → "street"（位置关系）
...

每个头学习不同的表示子空间。
```

**结构**：
```
MultiHead(Q, K, V) = Concat(head₁, head₂, ..., head₈)W^O

where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)

通常 h=8 个头，每个头 d_k=d_v=d_model/h=64 维。
```

#### Positional Encoding

**问题**：
```
没有 RNN，怎么知道词的顺序？

"The cat sat on the mat" ≠ "The mat sat on the cat"
```

**解决**：
```
在 Embedding 上加位置编码：

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

例：
词 1: Embedding₁ + PE(1)
词 2: Embedding₂ + PE(2)
...

这样模型就知道位置了。
```

**为什么用正弦/余弦？**
```
猜想：这样 PE(pos+k) 可以表示为 PE(pos) 的线性函数。

例：
PE(5) 和 PE(10) 的关系
= PE(0) 和 PE(5) 的关系

模型可以学习相对位置。
```

### 第三章：完整架构

#### Encoder-Decoder 结构

**Encoder**：
```
输入：x₁, x₂, ..., xₙ

处理：
1. Embedding + Positional Encoding
2. Multi-Head Self-Attention
3. Add & Norm
4. Feed-Forward
5. Add & Norm
6. (重复 6 层)

输出：z₁, z₂, ..., zₙ
```

**Decoder**：
```
输入：y₁, y₂, ..., yₘ

处理：
1. Embedding + Positional Encoding
2. Masked Multi-Head Self-Attention
   （只能关注之前的词）
3. Add & Norm
4. Encoder-Decoder Attention
   （Query 来自 Decoder，Key/Value 来自 Encoder）
5. Add & Norm
6. Feed-Forward
7. Add & Norm
8. (重复 6 层)

输出：概率分布 → 下一个词
```

#### 为什么这样设计？

**Encoder 的 Self-Attention**：
```
让每个词理解整个句子。

例：
"The bank was crowded"
"bank" 关注 "crowded" → 知道是"银行"不是"河岸"
```

**Decoder 的 Masked Attention**：
```
防止看到未来的词。

训练时：
输入："I love NLP"
不能让学生看到 "NLP" 再预测 "NLP"。

Mask 确保位置 i 只能看到位置 < i。
```

**Encoder-Decoder Attention**：
```
让 Decoder 关注输入。

例：
翻译 "我喜欢 NLP" → "I love NLP"

生成 "love" 时：
Query 来自 "love"
Key/Value 来自 "喜欢"

这样就知道对应关系。
```

### 第四章：实验结果

#### 实验 1：机器翻译（WMT 2014 En-De）

**数据集**：450 万句子对

**结果**：
| 模型 | BLEU | 训练时间 |
|------|------|---------|
| Best Previous | 26.2 | 几周 |
| Transformer | **28.4** | 3.5 天 |

**提升**：+2.2 BLEU（巨大提升）

#### 实验 2：机器翻译（WMT 2014 En-Fr）

**数据集**：3600 万句子

**结果**：
| 模型 | BLEU | 训练时间 |
|------|------|---------|
| Best Previous | 40.0+ | 几周 |
| Transformer | **41.8** | 3.5 天 |

**关键**：单个模型就超越之前的集成模型。

#### 实验 3：复杂度分析

| 层类型 | 每层复杂度 | 顺序操作数 | 最大路径长度 |
|--------|-----------|-----------|-------------|
| Self-Attention | O(n²·d) | O(1) | O(1) |
| Recurrent | O(n·d²) | O(n) | O(n) |
| Convolutional | O(k·n·d²) | O(1) | O(log(n)) |

**关键**：
- Self-Attention 的任意两词距离都是 O(1)
- RNN 需要 O(n) 步才能传递信息
- 这就是为什么 Transformer 能处理长依赖

### 第五章：预期 vs 实际

### 你的直觉 vs Transformer 的实现

| 维度 | 你的直觉/预期 | Transformer 实际 | 为什么 |
|------|--------------|-----------------|--------|
| **需要 RNN 吗？** | 需要 | 不需要 | 注意力足够捕捉依赖 |
| **需要卷积吗？** | 需要 | 不需要 | 注意力感受野是全局的 |
| **位置信息怎么办？** | 需要复杂机制 | 简单的正弦编码就够了 | 相对位置更重要 |
| **多头注意力必要吗？** | 可选 | 关键 | 捕捉多种关系 |
| **层数需要多少？** | 越多越好 | 6 层就够了 | 更深会训练困难 |

### 第六章：影响与局限

#### 影响

**直接影响**：
```
2017: Transformer 提出
2018: BERT (Encoder-only)
2019: GPT-2 (Decoder-only)
2020: GPT-3
2022: ChatGPT
2023: GPT-4, LLaMA, ...

所有大模型都基于 Transformer！
```

**间接影响**：
```
- 计算机视觉：ViT
- 语音处理：Whisper
- 多模态：CLIP, DALL-E
- 蛋白质折叠：AlphaFold

Transformer 成为通用架构。
```

#### 局限性

**局限 1：计算复杂度**
```
Self-Attention: O(n²)

序列长度 10 倍 → 计算量 100 倍！

长文本处理困难。
```

**局限 2：位置编码外推**
```
训练时最大长度 512。
测试时输入 1024 长度 → 效果下降。

正弦编码理论上可以外推，但实际不行。
```

**局限 3：缺乏归纳偏置**
```
RNN 有序列归纳偏置（天然处理顺序）。
CNN 有局部性归纳偏置。

Transformer 需要从数据学习一切。
→ 需要更多数据
```
