## 层 1：电梯演讲

**一句话概括**：Google 在 2018 年提出 BERT，通过双向 Transformer 编码器 + 掩码语言模型（MLM）+ 下一句预测（NSP）的预训练方式，首次实现"预训练 + 微调"范式，在 11 个 NLP 任务上刷新 SOTA，开启了 NLP 的预训练时代。

## 层 2：故事摘要

### 背景：2018 年 NLP 的"前 BERT 时代"

**时间**：2018 年初
**地点**：Google AI Language
**人物**：Jacob Devlin 团队

当时的局面：

```
预训练思想已经存在，但有两大阵营：

阵营 1：Feature-based（特征派）
代表：Word2Vec, GloVe, ELMo
思想：预训练词向量作为特征
缺点：
- 需要设计复杂的任务特定架构
- 特征冻结，无法适应任务
- ELMo 虽然上下文相关，但是浅层拼接

阵营 2：Fine-tuning（微调派）
代表：OpenAI GPT (2018.06)
思想：预训练整个模型，然后微调
缺点：
- 单向语言模型（从左到右）
- 无法看到"未来"的上下文
- 对需要双向上下文的任务不利
```

**核心问题**

> **能否结合两派优势，实现真正的"预训练 + 微调"范式，同时解决双向上下文的问题？**

### 研究者的洞察

想象 Devlin 团队的思考过程：

> "GPT 的微调范式很好，但为什么是单向的？"
>
> "因为语言模型天然就是单向的啊..."
>
> "等等，为什么一定要用语言模型的目标？"
>
> "如果把一些词遮住的，让模型预测被遮住的词呢？"
>
> "这样模型就能看到双向的上下文了！"

这就是 Masked Language Model (MLM) 的由来。

但还有一个问题：

> "句子之间的关系怎么学习？"
>
> "比如蕴含、问答，需要理解两个句子的关系"
>
> "再加一个任务：预测下一句！"

这就是 Next Sentence Prediction (NSP)。

### 技术路线图

```
┌───────────────────────────────────────────┐
│            BERT 架构                       │
├───────────────────────────────────────────┤
│ 输入表示                                  │
│  [CLS] 我 喜 欢 你 [SEP] 很 好 [SEP]        │

预训练任务：
1. MLM: 遮住 15% 的词，让模型预测
2. NSP: 给两个句子，预测 B 是否是 A 的下一句
```

### 第一章：研究者的困境

#### 1.1 预训练的演进历程

**阶段 1：静态词向量 (2013-2017)**

```
Word2Vec (2013):
思想：词义由上下文决定
方法：CBOW/Skip-gram
输出：每个词一个向量

问题：
- "我喜欢苹果"和"苹果发布了新手机"
- "苹果"的向量是一样的！
- 无法处理多义词
```

```
GloVe (2014):
思想：全局共现矩阵 + 局部上下文
改进：更好捕捉全局统计信息

问题：
- 还是静态向量
- 多义词问题未解决
```

**阶段 2：动态词向量 (2018)**

```
ELMo (2018.02):
思想：用 LSTM 编码上下文
改进：
- "苹果"在不同上下文有不同向量
- 上下文相关的表示

问题：
- 双向是浅层拼接（左→右 + 右→左）
- 需要设计任务特定架构
- 特征冻结，不能微调
```

**阶段 3：预训练 + 微调 (2018)**

```
OpenAI GPT (2018.06):
思想：用 Transformer decoder 预训练
改进：
- 预训练整个模型
- 下游任务微调

问题：
- 单向（从左到右）
- 看不到"未来"的上下文
- 对问答等任务不利
```

#### 1.2 双向的重要性

**为什么双向如此重要？**

看一个例子：

```
句子："bank"

左→右模型：
- 看到"I went to the"
- 预测"bank"
- 无法利用后面的"river"

右→左模型：
- 看到"the river"
- 预测"bank"
- 无法利用前面的"I went to"

双向模型（BERT）：
- 看到"I went to the [MASK] of the river"
- 理解是"河岸"而不是"银行"
```

**任务对双向的需求**

```
需要双向上下文的任务：
1. 问答
   - 问题："他在哪里出生？"
   - 需要看完整段上下文

2. 蕴含识别
   - 前提："猫在垫子上"
   - 假设："垫子上有东西"
   - 需要理解两句关系

3. 命名实体识别
   - "苹果发布了新产品"→ORG
   - "我喜欢吃苹果"→FOOD
   - 需要上下文消歧

不需要双向的任务：
1. 语言建模（生成）
   - 天然就是单向的

2. 文本分类（简单）
   - 整体语义即可
```

#### 1.3 研究问题

Devlin 团队在论文中提出的核心问题：

> **当前预训练方法的限制是什么？如何改进？**

具体限制：

```
1. 单向性限制
   - GPT 只能从左到右
   - 无法利用"未来"信息

2. 架构限制
   - 特征派需要复杂架构
   - 微调派受限于单向

3. 任务适应性
   - 不同任务需要不同设计
   - 缺乏统一框架
```

目标：

```
1. 实现真正的双向预训练
2. 简化下游任务流程
3. 在多个任务上验证
```

### 第二章：BERT 的设计

#### 2.1 核心创新：Masked Language Model

**问题：如何做双向语言模型？**

传统语言模型的定义：

```
P(w1, w2, ..., wn) = Π P(wi | w1, ..., wi-1)
                      ↑
                  只能看前面
```

这天然就是单向的！

Devlin 的洞察：

```
"我们为什么要用语言模型的目标？"

"语言模型是为了生成文本"
"但我们的目标是学习表示！"

"换个目标就好了！"
```

**Masked Language Model (MLM)**

```
原始句子：
"我喜欢吃苹果"

Mask 之后：
"我 [MASK] 吃 [MASK] 果"

任务：
预测 [MASK] 是什么

关键：
- 可以看到左右上下文
- "我 [MASK] 吃"→可能是"喜欢"
- "吃 [MASK] 果"→可能是"苹"
```

**具体实现**

```
Masking 策略：
- 随机选择 15% 的 token
- 80% 真正替换为 [MASK]
- 10% 替换为随机词
- 10% 保持不变

为什么这么复杂？
- 如果总是 [MASK]，测试时没有 [MASK]
- 保留一些不变，模型不能依赖 [MASK]
- 替换为随机词，增加鲁棒性
```

**示例**

```
输入："我喜欢吃苹果"

处理：
- 随机选到"欢"和"苹"
- "欢"→[MASK] (80% 概率)
- "苹"→"香" (10% 概率，随机词)

模型看到：
"我 [MASK] 吃香果"

任务：
- 预测 [MASK] = "欢"
- 预测"香"位置应该是"苹"
```

#### 2.2 Next Sentence Prediction (NSP)

**动机**

```
很多任务需要理解句子关系：
- 问答：问题 + 段落
- 蕴含：前提 + 假设
-  paraphrase：句子 1 + 句子 2

只靠 MLM 够吗？
- MLM 只学习词级别表示
- 句子级别关系学不到

需要额外任务！
```

**NSP 任务设计**

```
输入：句子 A + 句子 B

标签：
- IsNext: B 是 A 的下一句
- NotNext: B 不是 A 的下一句

构造数据：
- 正例：连续的两句
- 负例：随机抽的句子

任务：
二分类（IsNext or NotNext）
```

**示例**

```
正例：
A: "猫坐在垫子上"
B: "它很舒服"
标签：IsNext

负例：
A: "猫坐在垫子上"
B: "苹果是红色的"
标签：NotNext
```

#### 2.3 输入表示

**BERT 的输入格式**

```
[CLS] 我 喜 欢 你 [SEP] 很 好 [SEP]
 │     │               │
 │     │               └─ Sentence B
 │     └─ Sentence A
 └─ 分类 token

每个 token 的表示 = Token Emb + Segment Emb + Pos Emb
```

**特殊 token**

```
[CLS]:
- 放在句首
- 最终表示用于分类任务
- 聚合整个序列的信息

[SEP]:
- 分隔句子
- 单句输入也加在句尾
- 双句输入用在中间

[MASK]:
- 用于 MLM 任务
- 只在训练时出现
```

**Embedding 计算**

```python
# Token Embedding
token_emb = lookup(token_ids, vocab_emb)  # [seq_len, dim]

# Segment Embedding
seg_emb = lookup(segment_ids, seg_emb_table)  # [seq_len, dim]

# Position Embedding
pos_emb = position_emb_table[:seq_len]  # [seq_len, dim]

# 最终表示
embedding = token_emb + seg_emb + pos_emb
```

#### 2.4 模型架构

**BERT-Base**

```
层数：12 (Transformer blocks)
隐藏层维度：768
注意力头数：12
参数量：110M
```

**BERT-Large**

```
层数：24
隐藏层维度：1024
注意力头数：16
参数量：340M
```

**对比 GPT**

```
GPT (2018):
- 12 层，768 维，12 头
- Decoder-only (masked self-attention)
- 单向

BERT:
- 12 层，768 维，12 头
- Encoder-only (full self-attention)
- 双向
```

#### 2.5 预训练流程

```
数据：BooksCorpus (800M 词) + English Wikipedia (2500M 词)

步骤：
1. Tokenization (WordPiece)
2. 构造句子对
3. MLM masking (15%)
4. NSP 标签构造
5. 训练

硬件：4 个 TPU Pod (16 TPU v3)
时间：4 天
```

**训练细节**

```
Batch size: 256 (32 序列 × 8 TPU)
学习率：2e-5 (warmup + 线性衰减)
训练步数：1M 步
序列长度：128 (90%) + 384 (10%)
```

### 第三章：核心概念 - 大量实例

#### 3.1 双向 vs 单向

**生活类比 1：看书**

```
单向阅读（GPT）：
- 从左到右读
- 读到一半，不知道结局
- 无法理解伏笔

双向阅读（BERT）：
- 可以前后翻看
- 读到伏笔，可以往后看结局
- 理解更准确

例子：
"他拿起枪，指着目标，扣动了..."

单向：可能是"扳机"，也可能是"保险"
双向：看到后文"一声枪响"，确定是"扳机"
```

**生活类比 2：填字游戏**

```
单向填字：
_ _ _ T _ _ _
只能根据前面的字母猜

双向填字：
C _ _ T _ _ L
可以根据前后字母猜
答案是"COUNTRY"或"CAPITAL"
```

**代码示例**

```python
# 单向 Attention (GPT)
def causal_attention(q, k, v):
    scores = q @ k.T
    # Mask 掉未来
    mask = np.triu(np.ones_like(scores), k=1)
    scores = scores.masked_fill(mask == 1, -inf)
    weights = softmax(scores)
    return weights @ v

# 双向 Attention (BERT)
def bidirectional_attention(q, k, v):
    scores = q @ k.T
    # 没有 mask，可以看到所有位置
    weights = softmax(scores)
    return weights @ v
```

#### 3.2 MLM 任务

**生活类比：完形填空**

```
语文考试的完形填空：
"床前 [MASK] 月光，疑是地上霜"

根据上下文：
- 前面"床前"，后面"月光"
- 应该是"明月"或"的"

BERT 的 MLM 就是这样！
```

**训练示例**

```
原始输入：
"我 喜欢 吃 苹果"

随机 Mask：
- 选 15% 的 token
- "欢"被选中

80% 概率：
"我 喜欢 [MASK] 苹果"
目标：预测"欢"

10% 概率：
"我 喜欢 随机词 苹果"
目标：预测"欢"（知道随机词是错的）

10% 概率：
"我 喜欢 欢 苹果"
目标：预测"欢"（保持不变）
```

#### 3.3 NSP 任务

**生活类比：接龙游戏**

```
句子接龙：
A: "今天天气真好"
B: "我们去公园吧"→合理 (IsNext)
B: "香蕉是黄色的"→不合理 (NotNext)

NSP 就是让模型学会判断合不合理
```

**实际应用**

```
问答任务：
问题："张三毕业于哪所大学？"
段落 1："张三 1990 年出生"→NotNext
段落 2："张三毕业于清华大学"→IsNext (相关)

模型学到：
- 句子 2 与问题更相关
- 用 [CLS] 表示做相关判断
```

### 第四章：预期 vs 实际

#### 4.1 直觉预测 vs 实际结果

**预期 1：更大的模型 = 更好的效果**

| 你的直觉 | 实际 | 为什么 |
|---------|------|--------|
| Large 应该远超 Base | 提升有限 | 边际效应递减 |
| 更多层更好 | 12 层已经很好 | 深度增加训练难度 |
| 更大 batch 更好 | 适中最好 | 泛化能力问题 |

**预期 2：NSP 很重要**

| 你的直觉 | 实际 | 为什么 |
|---------|------|--------|
| NSP 是关键创新 | 作用有限 | MLM 已经学到句子关系 |
| 去掉 NSP 会大降 | 影响不大 | 后续研究证实 |
| NSP 越复杂越好 | 简单二分类足够 | 任务设计简洁 |

**预期 3：预训练数据越多越好**

| 你的直觉 | 实际 | 为什么 |
|---------|------|--------|
| 数据越多越好 | 质量更重要 | 噪声数据有害 |
| 需要百亿词 | 30 亿词足够 | 边际效应 |
| 多语言更好 | 单语先做好 | 语言间干扰 |

#### 4.2 反直觉挑战

**问题：如果 MLM 保留 10% 不变，模型会不会偷懒不看上下文？**

[先想 1 分钟...]

**直觉**：当然会！反正有些词不变。

**实际**：不会，因为模型不知道哪些不变。

```
解释：
- 模型不知道哪些词被 mask 了
- 每个词都可能是 mask
- 必须看上下文才能预测

这就是 BERT 的巧妙之处！
```

#### 6.1 方法对比

| 方法 | 双向 | 微调 | 架构复杂度 | SOTA 数量 |

#### 6.2 局限性

```
1. 独立性假设
   - MLM 假设被 mask 的词独立
   - 实际词之间有依赖

2. [MASK] 不匹配
   - 训练用 [MASK]
   - 测试没有 [MASK]

3. 计算成本
   - 预训练需要大量计算
   - 小团队难以复现

4. 序列长度
   - 最长 512 token
   - 长文档处理困难
```

#### 7.1 快速开始

### 总结

**BERT 的核心贡献**：

1. **双向预训练**
   - MLM 实现真正的双向
   - 超越单向语言模型

2. **简化流程**
   - 预训练 + 微调范式
   - 无需任务特定架构

3. **性能突破**
   - 11 个任务 SOTA
   - 首次超越人类（SQuAD）

4. **开源生态**
   - 代码模型开源
   - HuggingFace 集成
   - 数千后续工作

**一句话总结**：
> BERT 通过双向 Transformer + MLM + NSP 的预训练方式，开创了"预训练 + 微调"的 NLP 新范式，在 11 个任务上达到 SOTA，是 NLP 领域里程碑式的工作。
