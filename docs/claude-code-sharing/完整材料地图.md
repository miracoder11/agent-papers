## 二、可加入的番外/扩展

### 1. 机制与论文的对应关系

```
Agent Loop (s01) ────────────→ ReAct (2022)
     │                              "Reason + Act"
     │
     ↓
Tool Use (s02) ──────────────→ Toolformer (2023)
     │                              "模型学会调用工具"
     │
     ↓
验证闭环 ────────────────────→ Reflexion (2023)
     │                              "自我反思修正"
     │
     ↓
多智能体协作 (s09-s11) ──────→ MetaGPT, AutoGen (2023)
                                  "多角色协作"
```

### 2. 扩展想法（未来方向）

- **真正的长期记忆**：不只是 200K，而是跨会话的记忆
- **跨会话学习和进化**：每次用都变得更懂你
- **多 Agent 知识共享**：队友之间共享学到的东西
- **自我调试和自我修复**：出问题自己排查自己修
- **主动式常驻助手**：从"踹一下动一下"到"自己找活干"

### 3. 优化方向（实践技巧）

- **RTK（Rust Token Killer）**：减少 Tool Output 噪声
- **HANDOFF.md**：主动交接，不依赖压缩算法
- **Compact Instructions**：控制压缩行为
- **AI 审 AI**：一个写计划，一个审计划

## 三、多种讲述风格设计

### 风格 1：故事驱动型

**主线**：从用户困惑到理解设计的旅程

**特点**：
- 用"翻车现场"开场
- 每个设计决策背后有一个故事
- 像 AI 审 AI、Todo 工具变枷锁这种有趣细节
- 结尾有情感共鸣

**适合**：技术分享会、团队内部分享

### 风格 2：架构解析型

**主线**：六层架构作为核心框架

**特点**：
- 用六层架构组织全部内容
- 每层讲清楚"是什么、为什么、怎么用"
- 有数据支撑（12.5% 上下文开销）
- 有代码片段（30 行 agent loop）

**适合**：技术深度分享、架构评审

### 风格 3：实践指南型

**主线**：怎么用好 Claude Code

**特点**：
- 以"最佳实践"为核心
- 大量 actionable 建议
- 反模式警示
- 工程化布局参考

**适合**：团队培训、新手上路

### 风格 4：演化叙事型

**主线**：从 30 行代码到自治团队

**特点**：
- 按时间线讲演化
- 每个机制一句话格言
- 问题 → 解决方案 → 新问题的节奏
- 有种"追剧"的感觉

**适合**：技术科普、教学分享

### 风格 5：论文解读型

**主线**：从论文到工程实践

**特点**：
- 用论文作为学术背景
- 讲每个机制背后的学术思想
- ReAct → Agent Loop，Reflexion → 验证闭环
- 有理论深度

**适合**：学术分享、技术调研
